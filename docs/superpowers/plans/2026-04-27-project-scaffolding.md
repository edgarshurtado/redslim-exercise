# Project Scaffolding Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up three services (PostgreSQL in Docker, Django backend on port 8000, React frontend on port 8080) fully wired together and verifiable via a "Hello Redslim" page that calls the backend API.

**Architecture:** PostgreSQL runs in a Docker container exposed on `localhost:5432`. The Django backend runs locally, connects to Postgres, and exposes a single DRF endpoint at `/redslim-hello`. The React frontend runs locally on port 8080 and uses Axios to call that endpoint on page load, displaying the returned message.

**Tech Stack:** Docker Compose, PostgreSQL 16, Python/Django 5/DRF/django-cors-headers/Pytest, Node/React/Vite/MUI/Tailwind/Axios/Jest

```jsx
import { useState, useEffect } from 'react'
import { Typography, Container, Box } from '@mui/material'
import apiClient from './api/client'

function App() {
  const [message, setMessage] = useState(null)

  useEffect(() => {
    apiClient.get('/redslim-hello')
      .then(res => setMessage(res.data.message))
  }, [])

  return (
    <Container maxWidth="sm">
      <Box sx={{ mt: 8 }}>
        <Typography variant="h2" component="h1">
          Hello Redslim
        </Typography>
        {message && (
          <Typography variant="body1" data-testid="api-message">
            {message}
          </Typography>
        )}
      </Box>
    </Container>
  )
}

export default App
```
